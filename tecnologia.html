Por qué no uso frameworks innecesarios

En el desarrollo moderno, existe la tendencia a usar frameworks gigantes para resolver problemas pequeños. Esto añade complejidad, reduce el rendimiento y hace que los sistemas sean frágiles.

Mi enfoque es distinto: me baso en tecnologías estables y probadas. Si un problema puede resolverse con PHP y Vanilla JavaScript, no agrego una capa adicional de React o Node.js a menos que el proyecto realmente lo exija por motivos de interactividad o escala específica.

Arquitectura Modular

Uso Vertical Slicing. En lugar de dividir el código por capas técnicas (controladores, modelos, vistas), lo divido por características del negocio. Si falla el módulo de facturación, el resto del sistema no se entera.

Diseño de APIs REST

Las interfaces se comunican con el backend mediante APIs sólidas, predecibles y rápidas. Esto permite que el sistema crezca o se integre con terceros sin fricción.

IA Empresarial

Integración de agentes LLM mediante APIs de OpenAI o modelos locales. Utilizo la IA no como un adorno, sino como una herramienta cognitiva que procesa datos desestructurados.
